ObservabilityPack Studio

An engineering bench for canonical ObservabilityPack manifests — analyze a pack, compare your repo against live MCP signal, score against the maturity rubric, and compile to native Prometheus, OTel Collector, Alertmanager, and Grafana artefacts.

What it does

① ANALYZE
One pack, end to end
Browse by layer (L1–GOV), score conformance against the tier rubric, compile per artefact to the native platform format.
② COMPARE
Repo vs live, side by side
Diff two packs across every layer. Slice pills, atlas variants (Stratigraphy, Periodic Table, Constellation, Skyline, Transit, Arbor).
③ TRACEABILITY
What to do about the drift
Every artefact binned into Aligned / Declared-not-verified / Verified-not-declared / Stale. Per-row resolve and suppress, persisted locally.

Run it locally

The studio is an Express server plus a thin browser client. One install, one command — boots on http://127.0.0.1:8000.

# clone, install, run
git clone https://github.com/MoebiusX/otel-observability-pack-studio.git
cd otel-observability-pack-studio
npm install
npm run dev
# open http://127.0.0.1:8000 in a browser

Five reference packs ship under examples/ — payment-service (the spec's canonical tier-1 example), a tier-2 partial BAU, a tier-1 aspirational target, a tier-3 minimum, and a cron-refreshed live snapshot.

No hosted demo yet. Earlier studio versions were a single static HTML file we could publish on GitHub Pages; v0.3 split into a client + Node server for the live MCP fetcher, crawler, and conformance/compile endpoints. Hosting that needs a real Node runtime — coming after the current milestone.
